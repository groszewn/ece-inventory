API Guide (Meeseeks Inc.) :

Browsable API (through the Django Rest Framework) - all formats of responses returned in JSON

Viewing All Items

URL: https://meeseeksinc.colab.duke.edu/api/items/

GET request (seeing all items):

        Permissions: Admin, Manager, Users

        Fields:

item_name

quantity

model_number

description

tags

values_custom_field

Filtering (through GET request)

URL: https://meeseeksinc.colab.duke.edu/api/items/?item_name=&model_number=&quantity=&description=

        Permissions: Admin, Manager, Users

        Fields:

item_name

quantity

model_number

description

tags

        Note: click on the “Filters” button on the top right and choose what to filter with. Ctrl+Click to unselect tag, as well as selecting multiple tags to filter with.

POST request (adding a new item):

        Permissions: Admin, Manager

        Fields:

                item_name (required)

quantity (required)

model_number

description

tags (choose from existing tags - use tag ID)

values_custom_field

{

    "quantity": 10,

    "model_number": "1K Resistor",

    "description": 10,

Viewing Requests

URL: https://meeseeksinc.colab.duke.edu/api/requests/

GET request (viewing all requests):

Permissions: Admin / Manager (sees all requests), User (sees only the user’s requests)

        Fields:

                request_id

                user_id

                time_requested

                item_name

                request_quantity

                status

                comment

                reason

Viewing Request Detail

URL: https://meeseeksinc.colab.duke.edu/api/requests/<request_id>

GET request (seeing specific request):

        Permissions: Admin / Manager (sees request regardless of creator), Users (sees only the user’s own request)

        Fields:

item_name

quantity

model_number

description

tags

values_custom_field

PUT request (editing an existing requests):

        Permissions: Admin / Manager (can edit request regardless of creator), Users (can edit only the user’s own request)

        Fields that can be edited:

                requesty_quantity

reason

        Example:

{

    "request_quantity": 100,

    "reason": "yes please!"

}

DELETE request (deleting an existing request):

        Permissions: Admin / Manager (can delete request regardless of creator), Users (can delete only the user’s own request)

Creating a request

URL: https://meeseeksinc.colab.duke.edu/api/requests/create/<item_id>

POST request (creating a new request of an item):

        Permissions: Admin, Manager, User

        Fields:

request_quantity (required)

reason (required)

        Example:

                {

   "request_quantity":10,

   "reason": "I need this for ECE110"

}

Creating multiple requests

URL: https://meeseeksinc.colab.duke.edu/api/requests/multiple_create/<item_id>,<item_id>...   

POST request (creating new requests of items):

        Permissions: Admin, Manager, User

        Fields:

request_quantity (required)

reason (required)

        Example:

        [

    {

        "request_quantity": 1,

        "reason": "pls"

    },

    {

        "request_quantity": 2,

        "reason": "pls2"

    }

]

                Note: Input the number of JSONs for each item ID inputted in the URL (3 item IDs = 3 JSON dicts). If there are less IDs in the URL than JSON dicts in the POST request, it will create requests for the number of items in the URL.

How do I get an API Key?

Visit https://meeseeksinc.colab.duke.edu/ and login. On the top mid-right of the top bar, you should see a button that says “Get Access Token”. Use that to obtain your access token. Note that you only need the access token for using terminal-related API requests (such as curl or httpie)

Using Httpie :

The API was written so that anyone with correct authentication interact with the data on our server. Therefore, one can use something other than our front-end API debugger. Here, we will outline the usage of httpie. Since each URL / requests are outlined above, we will not rewrite all commands. However, we will list a few examples of how httpie could be used with the token authentication to submit requests. For more information on how to create requests with httpie, please refer to https://httpie.org/
