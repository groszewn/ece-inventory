API Guide (Meeseeks Inc.) :

Browsable API (through the Django Rest Framework) - all formats of responses returned in JSON

How To Use This Guide:

- Copy the provided url for each action into a browser to navigate to the relevant API page.

- Scroll to the bottom of the page and fill in the relevant fields or provide the necessary data for the changes you wish to make. (Commands requiring a raw data format have example data specified in the document.)

- Click the button at the bottom of the page to submit and execute your API command.

Creating a request

Creating multiple requests

URL: https://meeseeksinc.colab.duke.edu/api/requests/multiple_create/<item_id>,<item_id>...   

POST request (creating new requests of items):

        Permissions: Admin, Manager, User

        Fields:

request_quantity (required)

reason (required)

        Example:

        [

    {

        "request_quantity": 1,

        "reason": "pls"

    },

    {

        "request_quantity": 2,

        "reason": "pls2"

    }

]

                Note: Input the number of JSONs for each item ID inputted in the URL (3 item IDs = 3 JSON dicts). If there are less IDs in the URL than JSON dicts in the POST request, it will create requests for the number of items in the URL.

How do I get an API Key?

Visit https://meeseeksinc.colab.duke.edu/ and login. On the top mid-right of the top bar, you should see a button that says “Get Access Token”. Use that to obtain your access token. Note that you only need the access token for using terminal-related API requests (such as curl or httpie)

Using Httpie :

The API was written so that anyone with correct authentication interact with the data on our server. Therefore, one can use something other than our front-end API debugger. Here, we will outline the usage of httpie. Since each URL / requests are outlined above, we will not rewrite all commands. However, we will list a few examples of how httpie could be used with the token authentication to submit requests. For more information on how to create requests with httpie, please refer to https://httpie.org/
